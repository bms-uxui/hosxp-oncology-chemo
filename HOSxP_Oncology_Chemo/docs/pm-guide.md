## วิธีทดสอบระบบ

### เข้าสู่ระบบ
1. เปิด `localhost:5173/hosxp-oncology-chemo/`
2. กดปุ่ม **"เข้าสู่ระบบ"** → เข้าเป็น ONC_DOCTOR
3. เปลี่ยน Role ได้จากเมนูด้านล่างซ้าย (คลิกชื่อผู้ใช้)
4. PIN สำหรับทุก Role: **`123456`**

---

## Flow ที่ทดสอบได้ทั้งหมด

### Flow 1: ลงทะเบียนผู้ป่วยใหม่
**Role**: ONC_DOCTOR

1. หน้าหลัก → กดปุ่ม **"ลงทะเบียนผู้ป่วยใหม่"** (ที่ Banner)
2. **Step 1**: กรอก HN → กดค้นหา
   - HN ที่ทดสอบได้: `550123`, `660234`, `770345`
   - หรือกดปุ่ม HN ด้านล่างเพื่อกรอกอัตโนมัติ
3. **Step 2**: ข้อมูลผู้ป่วย (ซ้าย) จะแสดงอัตโนมัติ
   - กรอกข้อมูลการวินิจฉัย (ขวา):
     - **Diagnosis**: เลือกจาก dropdown
     - **ICD-10**: auto-fill จาก Diagnosis
     - **Morphology**: กรอกเอง
     - **Stage**: กดเลือก pill buttons (I → IVB)
     - **TNM**: กดเลือกใน 3 cards (T, N, M)
4. กด **"ลงทะเบียน"** → Skeleton Animation → Redirect ไปหน้าผู้ป่วย
5. ✅ Toast แจ้ง "ลงทะเบียน [ชื่อ] สำเร็จ"

---

### Flow 2: สั่งยาเคมีบำบัด (5 ขั้นตอน)
**Role**: ONC_DOCTOR

1. หน้าหลัก → คลิกผู้ป่วยในตาราง → เข้าหน้า Patient Summary
2. กด Tab **"สั่งยาเคมีบำบัด"**
3. ทำตาม 5 ขั้นตอน:

| Step | ชื่อ | สิ่งที่ต้องทำ |
|------|------|--------------|
| 1 | ประเมิน ECOG | เลือก ECOG 0-4 → กด "ถัดไป" |
| 2 | เลือก Protocol | กรองตามชนิดมะเร็ง → คลิกเลือก Protocol Card → กด "ถัดไป" |
| 3 | คำนวณขนาดยา | ระบบคำนวณอัตโนมัติ, ปรับ Dose Reduction ได้ → กด "ถัดไป" |
| 4 | กำหนดการให้ยา | เลือกวันนัด, Cycle, กรอก Clinical Notes → กด "ถัดไป" |
| 5 | ตรวจสอบ | Preview เอกสาร A4 → กด "ลงนาม" → กรอก PIN `123456` |

4. ✅ Toast แจ้ง "สร้างคำสั่งยา CHEMO-xxx สำเร็จ"
5. สามารถพิมพ์/ดาวน์โหลดเอกสารได้

---

### Flow 3: เภสัชกรตรวจสอบ + เตรียมยา
**Role**: ONC_PHARMACIST (เปลี่ยน Role ก่อน)

1. เมนู **"สูตรยาเคมีบำบัด"** หรือ Tab **"ตรวจสอบ/เตรียมยา"** ในหน้าผู้ป่วย
2. เลือก Order จากรายการซ้าย
3. ตรวจสอบรายการยา + Lab Safety
4. **อนุมัติ**:
   - กด **"ลงนามเตรียมยา"** → กรอก PIN `123456`
   - กรอก Lot No., Expiry, ปริมาณ, ยาเหลือทิ้ง
   - ระบบจะ:
     - สร้าง Billing Record อัตโนมัติ
     - ส่ง Notification ไป Nurse + Doctor
     - แสดง QR Sticker สำหรับพิมพ์ติดถุงยา
   - ✅ Toast แจ้ง "เตรียมยาเสร็จ"
5. **ปฏิเสธ**:
   - กด **"ปฏิเสธ"** → กรอกเหตุผล → PIN
   - ⚠️ Toast แจ้ง "ปฏิเสธคำสั่งยา"
   - Notification ส่งกลับ Doctor

---

### Flow 4: พยาบาลให้ยา
**Role**: CHEMO_NURSE (เปลี่ยน Role ก่อน)

1. เมนู **"บันทึกการให้ยา"** (หน้าจอเต็ม)
2. บันทึกการให้ยาข้างเตียง

---

### Flow 5: ค่าใช้จ่าย + ออกใบแจ้งหนี้
**Role**: BILLING_OFFICER (เปลี่ยน Role ก่อน)

1. เมนู **"ค่าใช้จ่าย"**
2. ดู Stat Cards: ทั้งหมด, รอตรวจสอบ, ออกใบแจ้งหนี้, รายได้รวม
3. คลิกรายการในตาราง → เข้าดูรายละเอียด
4. ตรวจสอบรายการค่าใช้จ่าย + สิทธิการรักษา
5. กด **"ยืนยันและออกใบแจ้งหนี้"** → PIN `123456`
6. ✅ Toast แจ้ง "ออกใบแจ้งหนี้สำเร็จ" → พิมพ์ใบแจ้งหนี้อัตโนมัติ

---

### Flow 6: จัดการสูตรยา (Regimen Master)
**Role**: ONC_PHARMACIST

1. เมนู **"สูตรยาเคมีบำบัด"**
2. **ดูรายการ**: Banner + Stat Cards + ตาราง Protocol
3. **ดูรายละเอียด**: คลิกแถวในตาราง
   - ซ้าย: เกี่ยวกับ Protocol, Stats 2x2, หมายเหตุ, ประวัติเวอร์ชัน
   - ขวา: Day Tabs + รายการยา (ProtocolCard style)
4. **แก้ไข**: กดปุ่ม "แก้ไข" ที่ Banner ขวาบน
   - แก้ไขชื่อ, รหัส, ชนิดมะเร็ง
   - ตั้งค่า Regimen (Cycle Length, Total Cycles, เจตนาการรักษา, ระดับอาเจียน)
   - เพิ่ม/ลบ Day, เพิ่ม/ลบยา (inline editable table)
   - กด "บันทึก" → Confirmation Modal → ✅ Toast
5. **สร้างใหม่**: กดปุ่ม "สร้าง Protocol ใหม่" ที่ Banner หน้าแรก
6. **คัดลอก**: กดปุ่ม "คัดลอก" → เข้า edit mode ทันที
7. **Validation**: ถ้ากรอกไม่ครบ → ❌ Error Toast แต่ละข้อ

## Toast Notifications

ระบบแสดง Toast (แจ้งเตือนมุมขวาล่าง) สำหรับทุกการดำเนินการสำคัญ:

| ประเภท | สี | ตัวอย่าง |
|--------|-----|---------|
| ✅ Success | เขียว | ลงทะเบียนสำเร็จ, บันทึก Protocol สำเร็จ, ออกใบแจ้งหนี้สำเร็จ |
| ❌ Error | แดง | กรุณากรอกชื่อยา, ขนาดยาต้องมากกว่า 0 |
| ⚠️ Warning | เหลือง | ปฏิเสธคำสั่งยา |

Toast จะหายไปอัตโนมัติหลัง 4 วินาที หรือกด ✕ เพื่อปิด

## ข้อจำกัดของ Prototype

1. **ไม่มี Backend** — ข้อมูลเก็บใน React State หายเมื่อ Refresh
2. **ข้อมูลเป็น Mock** — ผู้ป่วย 12 คน, Protocol 5 สูตร, Billing 1 รายการ
3. **PIN ไม่ปลอดภัย** — ใช้ `123456` สำหรับทุก Role
4. **ไม่มี Search Global** — ⌘K toggle state แต่ยังไม่มี UI
5. **ไม่มี Real-time** — Notification ไม่ push แบบ real-time
