ภาพรวมระบบ

HOSxP Oncology Chemo CPOE

ระบบ Computerized Physician Order Entry สำหรับการสั่งยาเคมีบำบัด ครอบคลุมกระบวนการตั้งแต่ลงทะเบียนผู้ป่วย → สั่งยา → ตรวจสอบ/เตรียมยา → ให้ยา → ออกใบแจ้งหนี้

React 19 TypeScript Tailwind v4 HeroUI Vite 7

Prototype — ระบบนี้เป็น Frontend MVP ไม่มี Backend จริง ข้อมูลเก็บใน React State และจะหายเมื่อ Refresh หน้า
PIN Code สำหรับทุก Role: 123456

บทบาทผู้ใช้งาน

ระบบรองรับ 5 บทบาท สลับ Role ได้จากเมนูด้านล่างซ้ายของ Sidebar

บทบาทชื่อแสดงสิ่งที่ทำได้
ONC_DOCTORนพ.สมชาย รักษาดีลงทะเบียนผู้ป่วย, สั่งยาเคมีบำบัด, ดูประวัติ, วางแผนการรักษา
ONC_PHARMACISTภก.วิไล ใจดีตรวจสอบคำสั่งยา, เตรียมยา, จัดการสูตรยา (Regimen Master)
CHEMO_NURSEพว.อรุณ แสงทองบันทึกการให้ยา, ติดตามสถานะผู้ป่วย
BILLING_OFFICERคุณสมศรี บัญชีตรวจสอบค่าใช้จ่าย, ออกใบแจ้งหนี้
ADMINAdmin ระบบจัดการค่าใช้จ่าย, ตั้งค่าระบบ

5.1 Global State (OncContext)

interface OncContextType {
  // Auth
  role: OncRole;
  setRole(r: OncRole): void;
  verifyPin(pin: string): boolean;  // accepts "1234" or "123456"

  // Order History
  orderHistory: OrderHistoryEntry[];
  addOrderHistory(entry): void;

  // Notifications
  notifications: OncNotification[];
  addNotification(n): void;
  markNotificationRead(id): void;
  unreadCount: number;

  // Billing
  billingRecords: BillingRecord[];
  addBillingRecord(r): void;
  updateBillingStatus(id, status, confirmedBy?): void;
}

5.3 Billing Auto-Generation

// เมื่อเภสัชกรลงนามเตรียมยาเสร็จ (PharmVerify.tsx)

// 1. คำนวณราคายาจาก DRUG_PRICES lookup
const drugItems = chemoDrugs.map(d => ({
  code: `DRG-${idx}`,
  description: `${d.name} ${d.finalDose}mg`,
  unitPrice: DRUG_PRICES[d.name],
}));

// 2. รวม SERVICE_ITEMS (ผสมยา 1,500฿ + ให้ยา 800฿ + พยาบาล 1,200฿)
const items = [...drugItems, ...diluentItems, ...SERVICE_ITEMS];

// 3. สร้าง BillingRecord → status: "PENDING"
addBillingRecord({ id, items, totalAmount, status: "PENDING" });

// 4. แจ้ง BILLING_OFFICER
addNotification({ type: "billing", targetRole: "BILLING_OFFICER" });

5.4 Key Data Types

interface Patient {
  hn: string;            // Hospital Number
  name: string;
  diagnosis: string;     // "Breast Cancer"
  icd10: string;         // "C50.9"
  stage: string;         // "IIIA"
  t, n, m: string;       // TNM staging
  ecog: number;          // 0-4
  weight, height: number;
  bsa: number;           // Body Surface Area (m²)
  crcl: number;          // Creatinine Clearance
  regimen: string;       // "CAF"
}

interface BillingRecord {
  id: string;            // "BIL-001"
  status: "PENDING" | "INVOICED";
  items: BillingItem[];
  totalAmount: number;
  coverageType: "UC" | "SSO" | "GOV" | "CASH" | "INS";
}

interface OncNotification {
  type: "rejection" | "approval" | "prepared" | "billing";
  targetRole: OncRole;   // ใครเห็น
  hn?: string;           // link ไปหน้าผู้ป่วย
}

Toast Notifications

ระบบแสดง Toast (แจ้งเตือนมุมขวาล่าง) พร้อม Slide-in Animation สำหรับทุกการดำเนินการสำคัญ

ประเภทสีตัวอย่าง
Successเขียวลงทะเบียนสำเร็จ, บันทึก Protocol สำเร็จ, ออกใบแจ้งหนี้สำเร็จ, สร้างคำสั่งยาสำเร็จ, เตรียมยาเสร็จ
Errorแดงกรุณากรอกชื่อยา, ขนาดยาต้องมากกว่า 0, ต้องมีเคมีบำบัดอย่างน้อย 1 ตัว
Warningเหลืองปฏิเสธคำสั่งยา
Toast หายไปอัตโนมัติหลัง 4 วินาที หรือกด ✕ เพื่อปิด — มี Slide-out Animation
